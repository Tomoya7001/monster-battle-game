## 1. エネミーシステム概要

### 1.1 基本方針

| 項目 | 内容 |
|------|------|
| 通常エネミー | 既存39体のプレイヤーモンスターを流用 |
| レアエネミー | **2体のみ新規作成** |
| ボス | 既存39体から選出（3体勝ち抜き型） |
| ステータス調整 | レベルと倍率で調整 |

### 1.2 設計思想

- **「このモンスター欲しい！」** → ガチャへの導線
- **敵の技・特性を学べる** → PvP準備になる
- **データ管理がシンプル** → 既存データを参照

```
【逃走判定】

- 高火力パーティで挑むのが有効
- 状態異常で行動妨害しても逃走は止められない
```

## 5. ステータス計算

### 5.1 通常エネミーのステータス

```
エネミーステータス = 基礎値 × レベル補正

レベル補正:
- Lv1-10: 基礎値そのまま
- Lv11-20: 基礎値 × 1.1
- Lv21-30: 基礎値 × 1.2
- Lv31-40: 基礎値 × 1.3
- Lv41-50: 基礎値 × 1.4
```

### 5.2 ボスのステータス

```
ボスステータス = 基礎値 × レベル補正 × 1.3（ボス補正）
```

### 5.3 HPの計算

```
エネミーHP = 52 + (Lv - 1) × 2 × ステータス補正

例: Lv30ボス
HP = 52 + 29 × 2 × 1.3 = 52 + 75.4 ≒ 127 → ボス補正で165程度
```
